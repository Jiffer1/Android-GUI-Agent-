移动端 GUI 自动化 Agent 技术方案介绍

一、方案目标
本方案面向多步手机界面自动化任务，目标是在保证稳定性的前提下完成以下能力：
1. 根据用户指令生成可执行步骤框架
2. 从当前截图中提取与任务相关的可交互 UI 元素
3. 结合当前界面与跨步进度信息输出下一步标准动作
4. 严格遵循评测接口格式，避免因格式问题导致失败

二、整体架构
当前实现采用三模块串联架构（单文件实现，避免复杂化）：
1. Planner（任务规划）
   - 输入：用户 instruction
   - 输出：app_name + subgoals（抽象子目标序列）
   - 作用：在 step=1 生成任务结构并触发 OPEN 动作

2. UIExtractor（界面结构化提取）
   - 输入：instruction + planner_subgoals + 当前截图
   - 输出：elements + page_type
   - elements 字段包含：text、type、point、selected、high_risk
   - 作用：只保留与任务相关、可交互且高价值的界面元素，控制输出规模（最多 20 个）

3. ActionAnalyzer（动作决策）
   - 输入：instruction、subgoals、ui_elements、page_type、previous_progress、当前截图
   - 输出：action + parameters + progress
   - 作用：根据当前状态决定下一步标准动作，同时生成简短进度摘要供下一轮使用

三、核心状态与跨步机制
Agent 内维护三类状态：
1. subgoals：Planner 产出的任务子目标序列
2. last_ui_state：上一轮解析出的界面结构（当前方案中保留）
3. progress：上一轮 ActionAnalyzer 输出的进度摘要

其中 progress 是关键上下文：
- 每一步都生成一句简短中文摘要
- 下一步通过 previous_progress 传入模型
- 用于减少重复操作（如重复打开筛选、重复进入已完成环节）

四、动作与接口规范
所有输出严格对齐 agent_base.py 的标准动作协议：
- CLICK: {"point": [x, y]}
- SCROLL: {"start_point": [x, y], "end_point": [x, y]}
- TYPE: {"text": "..."}
- OPEN: {"app_name": "..."}
- COMPLETE: {}

坐标统一使用 0-1000 归一化整数。

系统在 _normalize_schema 中进行统一规范化：
- 非法 action 兜底为 COMPLETE
- 参数缺失时补默认值并做坐标边界裁剪
- 保证传给 TestRunner 的格式稳定

五、稳定性设计
1. 模型原始输出日志
- 通过 _log_model 将 Planner/UIExtractor/ActionAnalyzer 每步原始文本落盘到 model_output.log
- 便于定位“模型说了什么”与“系统实际执行了什么”的差异

2. JSON 解析容错（关键改进）
- 主路径：_extract_json_object 尝试标准 JSON 解析
- 失败时：启用 _extract_action_fallback
  - 用正则提取 action
  - 用括号深度匹配提取 parameters 对象
  - 独立提取 progress
- 目的：避免模型输出轻微格式错误时误判为 COMPLETE

六、提示词策略
1. Planner
- 强调语义顺序、禁止同义重复、避免不必要拆分
- 对存在前后依赖的任务要求先后处理

2. UIExtractor
- 仅提取与 instruction/subgoals 相关元素
- 强调结构化候选组（如日期、选集、筛选项）的相邻信息保留
- 输出数量上限控制，降低噪声

3. ActionAnalyzer
- previous_progress 作为连续上下文优先参考
- COMPLETE 采用保守策略（需当前截图证据支撑）
- high_risk=true 元素默认禁止点击
- 强调按任务语义顺序推进，避免跳步

七、当前方案优势
1. 不增加新模块，结构清晰，易维护
2. 接口严格对齐评测规范，风险可控
3. 通过 progress 机制提升跨步一致性
4. 通过解析容错减少“误 COMPLETE”
5. 日志可追踪，便于快速回归定位

八、后续可持续优化方向
1. 进一步优化 UI 元素筛选精度，降低无关项干扰
2. 优化 ActionAnalyzer 在边界页面的完成态识别
3. 针对高频失败样例迭代提示词细节，提高泛化稳定性
